# Upwork Freelance Automation Suite

Scrapes Upwork for relevant gigs, scores them against your profile, and generates tailored proposals for manual submission.

**You always submit manually.** This suite handles discovery and drafting — never form interaction.

---

## First-Time Setup (do this once)

### 1. Install dependencies

```bash
cd /path/to/freelance-automation-suite
npm install
```

Requires Node 20+.

### 2. Create your Supabase project

- Go to [supabase.com](https://supabase.com) and create a free project
- Run `mcp-server/schema.sql` in the Supabase SQL editor to create the tables
- Copy your project URL and service role key

### 3. Set environment variables

```bash
cp .env.example .env
```

Open `.env` and fill in:

```
UPWORK_SUPABASE_URL=https://your-project.supabase.co
UPWORK_SUPABASE_KEY=your-service-role-key
```

### 4. Deploy the MCP server

Deploy `mcp-server/` as a Supabase Edge Function and register it as `upwork-hunt` in your Claude config. See `docs/mcp-setup.md` for step-by-step instructions.

### 5. Run the setup interview

Open this project folder in Claude (Cowork or Claude Code), then run:

```
/setup
```

Claude will interview you about your niche, work history, skills, rate, and target gigs. At the end it writes five files:

| File | Purpose |
|------|---------|
| `USER_CONFIG.md` | Your niche, search queries, scoring keywords, rate |
| `playwright/user-config.js` | JS mirror used by the scraper and scorer |
| `MASTER_PROFILE.md` | Your Upwork positioning and headline |
| `MASTER_PROPOSALS.md` | Proof snippets, hooks, voice notes |
| `MASTER_SKILLS.md` | Skills rated Confident / Comfortable / Familiar |

### 6. Save your Upwork session

```bash
node playwright/save-session.js
```

A browser window opens. Log in to Upwork manually (including 2FA). Once you land on the jobs page, the script saves your session and closes. You won't need to do this again unless the session expires.

## Daily Usage

### Find today's gigs

```
/find-gigs
```

Scrapes your configured search queries, scores each gig, inserts new ones to the database, and shows a digest of HIGH and MEDIUM priority gigs.

### Write a proposal

```
/write-proposal <gig-url>
```

or

```
/write-proposal <gig-id>
```

Pulls the gig details, matches against your knowledge base, and generates a ready-to-paste proposal (plain text, 150–250 words). You copy it and submit on Upwork yourself.

## Scheduled Jobs (optional but recommended)

Set these up to run automatically:

| Time | Command | Purpose |
|------|---------|---------|
| 22:00 nightly | `node playwright/check-session.js` | Checks session health. Sends a macOS notification if re-auth is needed before the morning scrape. |
| 06:30 daily | `/find-gigs` | Runs the scraper across all search queries |
| 08:30 daily | digest | Shows new HIGH/MEDIUM gigs and pipeline status |

## Key Constraints

- **Read-only scraping** — Playwright never fills a form or clicks a submit button
- **Manual submission only** — you copy the proposal text and paste it into Upwork yourself
- **Session file is gitignored** — `playwright/session.json` never gets committed
- **`.env` is gitignored** — credentials never get committed
- **Max 10 searches per day** — tracked in `playwright/state.json` to stay under Upwork's radar

---

## Sharing / Cloning

This project has no hardcoded personal data. Anyone can clone it, run `/setup`, and configure it for their own freelance niche. All user-specific values live in `USER_CONFIG.md` and `playwright/user-config.js`, both generated by the setup interview.
